# 📚 Руководство по параметрам Paint by Numbers

Этот документ объясняет "физику" и математику пайплайна генерации.

## 1. Предобработка (Подготовка "глины")
Цель этапа: убить лишние градиенты и шум до того, как алгоритм начнет считать цвета.

### Метод сглаживания (filter_type)
- **pyrMeanShift (Пирамидальное среднее смещение):** Работает как гравитация для цветов. Скатывает пиксели с похожими оттенками в единые плоские цветовые "ямы" (террасы). Идеально для картин по номерам, делает из фото векторный мультик.
- **bilateral (Двусторонний фильтр):** Умное размытие. Смешивает соседние пиксели, но принципиально отказывается смешивать те, что сильно отличаются по цвету. Делает кожу гладкой, но сохраняет ресницы и контуры лица абсолютно резкими. Выбор для портретов.
- **median (Медианный фильтр):** Строит соседей по росту и берет среднего. Игнорирует резкие выбросы (шум "соль и перец"). Делает края немного ступенчатыми.

### Карта внимания (Saliency Map)
Ищет высокочастотные детали. На этапе *постобработки* меняет правила: позволяет сохранять крошечные пятна краски там, где алгоритм увидел "важный объект", и безжалостно сливает пятна в гигантские лужи на "скучном фоне".

## 2. Квантование (Выбор красок и форм)
Цель этапа: выбрать N баночек с краской и наложить их на холст.

### Метод палитры (palette_method)
- **K-Means:** Ищет среднее арифметическое масс цветов. Быстро, хорошо для пейзажей. Может "потерять" мелкие уникальные детали (зрачок глаза), если их площадь мала.
- **Median Cut / Octree:** Дробит цветовой куб. Гарантированно выделяет цвет под экстремальные оттенки, даже если их на фото всего 10 пикселей.
- **Взвешенный K-Means:** Находит контуры (Canny) и искусственно умножает их вес в X раз. Алгоритм вынужден выделить краску для тонких линий. Спасает портреты.

### Метод пространственной сегментации (spatial_method)
- **none:** Пиксели красятся независимо.
- **SLIC (Superpixels):** Рубит холст на ровные соты-мазки в 5D-пространстве (X,Y,L,a,b). Дает эффект "мазков кистью".
- **Watershed (Водораздел):** Считает контуры горами, а градиенты - долинами. "Затапливает" долины цветом. Дает очень органичные пятна, облегающие реальные формы объектов.

---

## 3. Постобработка (Контроль качества)
Физически удаляет мазки краски, которые невозможно будет напечатать или закрасить кисточкой.

- **Выращивание деталей (Grow Details):** Бинарная морфологическая дилатация. Если контрастная деталь (например, блик в глазу) оказалась слишком маленькой, алгоритм не удалит ее, а будет искусственно "раздувать" во все стороны за счет соседей, пока она не станет достаточно большой для печати.
